Programa desarrollado para la materia de Teoria de la computación Aguilar Garcia Rafael.

Notas:
  • 1.- Con cualquier separador regresa a q0 a menos que esté especificada una transición diferente
  • 2.- Si se ingresa algun caracter no establecido en las transiciones de un estado a otro, el automata llegara a un estado muerto
  • q0: Estado inicial y estado de reposo.
  • q1: Estado para manejar operadores + y -.
  • q2: Estado para manejar números decimales.
  • q3: Estado para manejar el número 0 y números octales.
  • q4: Estado para manejar números hexadecimales.
  • q5: Estado para manejar exponentes en números decimales.
  • q6: Estado para manejar el operador !.
  • q7: Estado para manejar el operador *.
  • q8: Estado para manejar exponentes en números decimales.
  • q9: Estado para manejar exponentes en números decimales.
  • q10: Estado para verificar el cierre de comentarios de bloque.
  • q11: Estado para manejar ++ o --.
  • q12: Estado para manejar el operador /.
  • q14: Estado para manejar el operador %.
  • C0: Estado para manejar números enteros.
  • C1: Estado para manejar números hexadecimales.
  • C2: Estado para manejar números octales.
  • C3: Estado para manejar números decimales con parte fraccionaria.
  • C4: Estado para manejar exponentes en números decimales.
  • C5: Estado para manejar identificadores.
  • C6: Estado para manejar comentarios de línea.
  • C6_1: Estado para manejar comentarios de bloque.
  • C7: Estado para manejar palabras reservadas.
  • C8: Estado para manejar el operador =.
  • C8_1: Estado para manejar operadores < y >.
  • C8_2: Estado para manejar operadores de comparación.
